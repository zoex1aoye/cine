# Source quality API audit

Generated by `dart run tool/audit_video_detail.dart`.

## Findings

### Field names

- Regular movies expose **both** `vip_source_list_source` and `source_list_source`.
- Short drama uses `playlist[]` with `source_config_name` (often `常规线路`).
- Parser now merges `vip_source_list_source` first, then `source_list_source`.
- Item-level `source_config_name` is preserved on `VideoSource.sourceConfigName`.

### Observed group names (movie id=98371)

```
VIP线路, 极速蓝光, 高速蓝光, 蓝光线路,
高清线路1, 高清线路2, 高清线路4,
LZ线路, SN线路, MT线路, … (CDN mirrors)
```

**Important:** `name` is **not** only clarity tiers — it mixes quality groups and CDN line mirrors. Quality classification uses keyword matching (`高清`, `蓝光`, `标清`, `流畅`, `*线路`).

### Multi-tier per episode

Same episode can appear under VIP线路 + 极速蓝光 + 高速蓝光 (3 URLs).

### Short drama

Single tier `常规线路` per episode — quality picker falls back to latency-only.

## Client strategy

| Player | Selection |
|--------|-----------|
| Main | Latency pool (≤500ms, then ≤800ms) → **高清 > 蓝光/VIP > 标清 > 流畅** → tie-break by ms |
| Preview | **流畅 / 标清** preferred, then lowest ms |

## Go decision

**Go** — multi-tier independent URLs confirmed; quality-aware pick is feasible with label heuristics.
